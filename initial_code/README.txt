#  INSTRUCTIONS  #
##################

* Go to project > website > database
* From command line, run: sh build_database_and_geo.sh
* Go to project > website
* From command line, run: python manage.py migrate
* From command line, run: python manage.py runserver
* In web browser, navigate to: http://127.0.0.1:8000/nmatch/home

#  OUTPUT  #
############

Documentation: Original

Description: Files for database build; all but output_census_tractsX.XX.csv will be regenerated if you build database

Select file descriptions:

* initial_output.csv: Consolidated raw data by census tract       
* output_census_tracts1.25.csv: Census tracts with nearby census (1.25 km radius)
* output_census_tracts1.5.csv: Census tracts with nearby census (1.5 km radius)
* output_census_tracts1.csv: Census tracts with nearby census (1 km radius)
* output_census_tracts2.csv: Census tracts with nearby census (2 km radius)
* output_map_info.py: Code to pull info from tracts database for map info windows
* score_columns.json: Columns to be scored
* standardized_scores.csv: Consolidated scored data by census tract

######################
#  WEBSITE/DATABASE  #
######################

Documentation: Original

Description: Files to build database on website

Select file descriptions:

* __init__.py: Creates database package
* bordering_census.py: Creates csv of census tracts and, for each tract, nearby tracts
* build.py: Write SQL statements to create raw_tract_data, multiple_tracts, and nearby_tracts tables
* build_base_tables.sql: Will be regenerated by build.py if you build database
* build_database_and_geo.sh: Script to build tracts database
* geocoding.py: Looks up census tracts of locational records in csvs 
* import_files.py: Creates tract data csv for tracts database
* standardize.py: Creates csv of tract scores and SQL code to create score_table table
* standardize.sql: Will be regenerated by standardize.py if you build database
